File I/O and Sockets Reference

Quick reference for Module 7 topics: Unix file descriptors, buffered I/O, directories, and TCP socket programming.

FDName

Low-Level File I/O

int fd = open(path, O_RDONLY);
ssize_t n = read(fd, buf, sizeof(buf));
ssize_t m = write(fd2, buf, n);
close(fd);
  • open returns -1 on error.
  • read returns 0 at EOF.
  • write may perform partial writes; loop until all bytes are written.

Open Flags

open("out.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
FlagMeaning
O_RDONLYOpen for reading only
O_WRONLYOpen for writing only
O_RDWROpen for read/write
O_CREATCreate if missing (requires mode)
O_TRUNCTruncate existing file
O_APPENDAppend writes at end

Buffered FILE* I/O

FILE *fp = fopen("in.txt", "r");
int c = fgetc(fp);
fclose(fp);
  • FILE* is buffered in user space; usually faster than byte-at-a-time read.
  • Use fflush(fp) to force buffered output to be written.

Directory Iteration

DIR *d = opendir(".");
struct dirent *ent;
while ((ent = readdir(d)) != NULL) {
    // ent->d_name
}
closedir(d);

TCP Socket Lifecycle (Server)

int listenfd = socket(AF_INET, SOCK_STREAM, 0);
setsockopt(listenfd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));
bind(listenfd, (struct sockaddr *)&addr, sizeof(addr));
listen(listenfd, backlog);
int connfd = accept(listenfd, (struct sockaddr *)&client, &clen);

Order is important: socket -> bind -> listen -> accept.

TCP Client Lifecycle

int fd = socket(AF_INET, SOCK_STREAM, 0);
connect(fd, (struct sockaddr *)&server, sizeof(server));
write(fd, req, req_len);
read(fd, buf, sizeof(buf));
close(fd);

Address Conversion

struct sockaddr_in a;
memset(&a, 0, sizeof(a));
a.sin_family = AF_INET;
a.sin_port = htons(8080);
inet_pton(AF_INET, "127.0.0.1", &a.sin_addr);
  • htons/htonl: host -> network byte order.
  • ntohs/ntohl: network -> host byte order.
  • inet_pton: parse text IP to binary form.

Partial Read/Write Rules

  • TCP is a byte stream; message boundaries are not preserved.
  • One write() can be observed as multiple read() calls.
  • Loop until protocol-level message is complete.

Threaded Server FD Handoff Pattern

int client_fd = accept(...);
int *arg = malloc(sizeof(*arg));
*arg = client_fd;
pthread_create(&tid, NULL, handle_client, arg);
pthread_detach(tid);

Passing &client_fd directly from loop stack variable can race and hand wrong FD to threads.

HTTP Mini-Server Essentials

  • Request line: method, path, version.
  • Response starts with status line, then headers, blank line, then body.
  • Set at least Content-Length and Content-Type.
  • Path sanitization: reject .. to prevent directory traversal.

Common Mistakes Checklist

  • Ignoring partial writes/reads on sockets.
  • Forgetting to close both connection FDs and listening FDs where appropriate.
  • Calling bind without SO_REUSEADDR and hitting restart delays.
  • Using host byte order directly in sin_port.
  • Not checking return values (-1) and errno.
