Virtual Memory, Linking, and Heap Reference

Quick reference for Module 9 topics: address translation, linking, heap allocation, Valgrind, and garbage collection concepts.

Virtual Memory Basics

  • User processes use virtual addresses; MMU translates to physical addresses.
  • TLB caches recent translations.
  • With 4 KB pages, offset uses 12 low-order bits.
  • PTE fields commonly include valid bit, physical page number, and permissions.

Page Fault Flow

  1. Access hits invalid or non-resident PTE.
  2. CPU raises page fault exception.
  3. Kernel handler validates access and resolves mapping/load.
  4. Instruction is retried or process is signaled (for invalid access).

Static vs Dynamic Linking

ModeTrait
StaticLibrary code copied into executable (larger binary)
DynamicLibraries resolved at load/runtime (smaller binary, shared libs)
 
$ gcc main.c -o dyn_app
$ gcc -static main.c -o static_app
$ ldd dyn_app

PLT and GOT (Dynamic Linking)

  • PLT: call stubs for external functions.
  • GOT: runtime-resolved addresses used by those stubs.
  • First call may trigger resolver; later calls usually jump directly via GOT.

Symbol Strength (Common Rules)

  • Strong symbols: functions, initialized globals.
  • Weak/common behavior for tentative globals is toolchain-dependent (historically common with -fcommon).
  • If one strong and one weak/common definition collide, the strong definition is selected.

Heap Allocation APIs

void *p = malloc(n);
void *q = calloc(count, size);
void *r = realloc(p, new_n);
free(p);
  • malloc returns NULL on failure.
  • calloc allocates and zero-initializes.
  • realloc may move block; always use returned pointer carefully.
  • Double-free and use-after-free are undefined behavior.

Fragmentation Terms

  • Internal fragmentation: wasted bytes inside allocated blocks.
  • External fragmentation: free memory exists but split into unusable pieces.
  • Coalescing: merging adjacent free blocks.

Valgrind Quick Commands

$ gcc -g app.c -o app
$ valgrind --leak-check=full ./app
$ valgrind --track-origins=yes ./app

Garbage Collection Concepts

  • Reference counting fails on cycles unless cycle detection is added.
  • Mark-and-sweep: mark reachable from roots, then sweep unreachable.
  • Generational GC collects young objects frequently and old objects less often.
  • C's unrestricted pointer arithmetic makes moving GC difficult.

Common Mistakes Checklist

  • Forgetting to free on all paths.
  • Using pointers after free.
  • Writing one byte past array bounds.
  • Assuming linking errors are compiler errors; often missing objects/libraries.
  • Ignoring page-fault/performance effects of access patterns.
