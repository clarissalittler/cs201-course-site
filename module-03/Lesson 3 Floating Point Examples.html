Lesson 3: Floating Point Examples

Floating Point Examples

Here are a few floating point examples in single-precision. Double would be treated the same except with more bits in the exponent and significand fields and a different bias (1023).

These mainly show going from the memory's bit pattern to a fraction binary number (or a special case) and vice versa.

From bit pattern to fractional binary number

Example 1 (I'm using spaces between the fields for clarity):

    1 01101111 01101100000000000000000

The first bit is 1 so this is a negative number. The exponent field is decimal number 0*128 + 1*64 + 1*32 + 0*16 +1*8 + 1*4 + 1*2 + 1*1 = 64 + 32 + 8 + 4 + 2 + 1 = 111 (You can also use a calculator to do the conversion). Now this is the bit pattern, so to get back the exponent, we need to subtract the bias of 127 from it: 111 - 127 = -16. Note this works with the left-most bit being 0 (negative).

So now all we need is to piece the parts together to get the fractional binary representation (remember the extra 1 in front for the significand part):

    -1.011011 x 2-16

Keep in mind that 2 and -16 are decimal equivalents of the actual binary digits. The significand part is in binary.

Example 2:

    0 00000000 01011100000000000000000

The sign bit indicates this is a positive number. Note the exponent field is all zeros. This is a denormalized case. The exponent is -126, and the significand doesn't have a leading 1:

 .010111 x 2-126 or 1.0111 x 2-128

Example 3:

    0 11111111 10011100000000000000000

Note the exponent field is all ones. This is a special case. Since the significand is not all zeros, this is a NaN. It's also a quiet NaN because there is a 1 in the leftmost bit of the significand.

From Fractional Binary to Bit Pattern

Example 2: Find the bit pattern for this fractional binary number:

  +1.0001 x 230

Since this is a positive number, the sign bit is 0. To get the exponent bits, take the bias and add it to the exponent: 127+30 = 157. The bit pattern for this is 10011101 (Note: I used a calculator. Students can use one on quizzes as well). For the significand bits, drop off the first 1. so the bit pattern is 000100000000000000000000. So, putting the fields together gives:

   010011101000100000000000000000000
