Module 3 Summary

Wrap up

In this module, you wrote x86-64 assembly programs by hand using the as and ld toolchain, and compared your programs with the output of gcc -S. You progressed from a 3-line exit program to a complete program that reads input, computes results using arithmetic and stack operations, and prints output.

Key Takeaways

  • The build process has four stages: preprocessing, compiling, assembling, and linking. You can write assembly directly in .s files and use as and ld to bypass the compiler entirely.
  • x86-64 processors have 16 general-purpose registers, each accessible at 64-bit, 32-bit, 16-bit, or 8-bit widths. AT&T syntax uses % for registers, $ for immediates, and size suffixes (b/w/l/q).
  • The mov instruction copies data between registers and memory but cannot move memory-to-memory. The lea instruction computes addresses and is also used for fast arithmetic.
  • Global data lives in .data (read-write), .rodata (read-only), and .bss (zero-initialized) sections. Data is accessed using RIP-relative addressing: label(%rip).
  • Arithmetic instructions (add, sub, imul, idiv) and logical/shift instructions (and, or, xor, shl, shr) operate on registers and memory. Division uses implicit %rdx:%rax operands and requires cqto for sign extension.
  • The program stack grows downward. push/pop and manual sub/add on %rsp provide temporary storage. Multi-file programs use .global/.extern and link with ld.
