Module 3 Overview

In Module 3: x86-64 Assembly Fundamentals, you will learn to write x86-64 assembly programs by hand. This is something that most students have not encountered in their previous programming experience, and it is one of the most important topics in computer systems.

We start by examining the build process and writing your first assembly program using the GNU tools as and ld. From there, we cover registers, data movement and addressing modes, the data section and labels, arithmetic operations, logical and shift operations, and stack operations. Each topic follows a "write it by hand first, then see how gcc does it" approach: you write a working assembly program, then compare it with gcc -S output to understand what the compiler generates.

Learning objectives

At the end of this learning module, you will be able to:

  1. Describe the four-stage build process and the role of the compiler, assembler, and linker
  2. Use as and ld to assemble and link hand-written assembly programs
  3. Write Makefiles to automate the assembly and linking of single-file and multi-file projects
  4. Use gcc -S and objdump -d to examine compiler-generated assembly
  5. Identify x86-64 registers, their sub-register names, and their typical roles
  6. Write programs that use the .data, .rodata, and .bss sections with RIP-relative addressing
  7. Use the mov and lea instructions with all addressing modes
  8. Write programs using arithmetic instructions (add, sub, imul, idiv) and logical/shift instructions
  9. Use push, pop, and manual stack manipulation for temporary storage
  10. Write a complete assembly program that reads input, computes a result, and prints output

Learning activities

You are expected to do the following tasks in order to achieve the above objectives:

  1. Complete the reading listed below
  2. Read Lessons 1 to 10, work through the hand-written assembly examples, and complete the embedded practice problems
  3. Take the module quiz
  4. Review additional resources on the Module 3 Summary page
