Module 7 Overview

In Module 7: File I/O and Sockets Programming, we will explore how Unix systems handle input and output at the lowest level and then build on that foundation to write networked programs. Unix is built on a powerful idea: everything is a file. Whether you are reading from a disk, writing to a terminal, or communicating over a network, you interact with the same set of system calls. This module traces that idea from opening a file on disk all the way to building a working web server.

We begin with low-level file I/O using the system calls open, read, write, and close. We then examine the buffered FILE* API provided by the C standard library and learn how to work with directories and file metadata. In the second half of the module, we turn to networking. We study the TCP/IP protocol stack, learn the sockets API that lets programs communicate over a network, and progressively build more capable servers: first an echo server, then a multi-client threaded server, and finally a minimal HTTP web server that ties together file I/O, sockets, and threads.

This module is the capstone of the course. It brings together nearly every concept you have learned: C programming, system calls, processes, and threads. By the end, you will see how a few simple abstractions -- file descriptors and byte streams -- underlie all of modern networked computing.

Learning objectives

At the end of this learning module, you will be able to:

  1. Explain the Unix "everything is a file" philosophy and describe what a file descriptor represents
  2. Use the low-level I/O system calls open, read, write, and close to manipulate files
  3. Compare and contrast unbuffered I/O (file descriptors) with buffered I/O (the FILE* API)
  4. Read directory contents and query file metadata using opendir, readdir, and stat
  5. Describe the four layers of the TCP/IP model and explain what TCP provides to applications
  6. Explain the purpose of IP addresses, ports, and network byte order
  7. Implement a TCP server using the socket API: socket, bind, listen, accept
  8. Implement a TCP client using socket and connect
  9. Build a multi-client server using threads
  10. Construct a minimal HTTP web server that serves static files
