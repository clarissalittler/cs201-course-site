Module 7 Summary

Wrap up

This week we discussed Flow Control and Exceptional Flow Control.  We talked about exceptions, what causes them, and how they are handled.  We also learned about Processes and how these processes affect Exceptions in a Linux system. 

Process control goes over the use of some popular library functions for creating and managing processes. We also looked at several examples of using Fork to create child processes.

Key Takeaways

  • The sole job of the CPU is to process one instruction after another. Going from one instruction to another is called control transfer. A set of control transfers is called a flow of control or control flow.
  • The events that cause an EFC to happen are called generically exceptions. An exception is an abrupt change in the control flow in response to some change in the processor’s state.
  • In modern Linux systems, there are 256 possible exceptions. Numbers 0-31 are hardware exceptions defined by the architecture (Intel x86-64), and numbers 32-255 are interrupts and traps defined by the operating system (Linux).
  • According to the textbook, "A process is the instance of a program in execution." A program is a series of instructions that are run as a process (or as several processes as a program can be executed several times concurrently).
