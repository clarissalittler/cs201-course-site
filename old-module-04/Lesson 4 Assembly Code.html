Lesson 4: Assembly Code

Assembly Code

Each assembly code line starts with an instruction (movq, etc.) that tells the CPU what to do. What follows the instruction is one, two, or possibly three operands. These basically tell the CPU what the data is or where the data is stored.

For example, from the multstore.s file:

movq %rdx, %rbx

Here, the instruction is movq, and the two operands are %rdx and %rbx. Basically, this tells the CPU to move a value from the register rdx to the register rbx.

Data Size Indicators

Most instructions end in a one-letter data size indicator that tells the data sizes of the operands. Below is a table (from Figure 3.1 in the textbook) of the one-letter indicators:

Assembly code suffix

 Intel data type Sizes (bytes) C declaration
b Byte 1 char
w Word 2 short
l Double Word 4 int
q Quad Word 8 long or any pointer

I have only included integer types as we will be discussing integer data for a while. We will talk about floating point types later.
