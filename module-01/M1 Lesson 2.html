Numeric Representation: Hexadecimal Numbers

Binary numbers are a bit unwieldy to read. Instead of trying to display binary information in binary digits, a more compact numbering system related to binary is used to display binary numbers. This is the hexadecimal system.

The hexadecimal system is a numeric system with 16 digits. The digits are represented by the digits 0 to 9 and the letters A to F. The letters are needed to add the extra digits past 9 that a decimal system doesn't have. 'A' represents the equivalent of the decimal number 10, 'B' represents the equivalent of decimal 11, and so on.

Each hexadecimal digit can be directly translated into a set of 4 binary digits. Below is a table of each hex digit and its corresponding 4 binary digits:

Hex Digit Binary Number
0 0000
1 0001
2 0010
3 0011
4 0100
5 0101
6 0110
7 0111
8 1000
9 1001
A (10) 1010
B (11) 1011
C (12) 1100
D (13) 1101
E (14) 1110
F (15) 1111

Since translating between hexadecimal and binary is an exact one-digit to four-digit translation, it's easy to convert between the two numbering systems. For example, if a hex number is:

   98A3F

then, the binary equivalent is obtained by substituting each hex digit for its equivalent 4 bit binary number:

    9    8    A    3    F
  1001 1000 1010 0011 1111

So, the equivalent binary number is 10011000101000111111. Note that the hex number only requires 5 digits, and the binary number requires 20 digits. This makes for a more compact representation in hex and is the reason why it is common to display binary data as hex digits.

Although this is rarely done, you can translate hex numbers into decimal numbers. This is not an exact digits-to-digits translation. To go from hex to decimal, use the formula:

    decimal = h(n)*16n + h(n-1)*16(n-1) +...+ h(1)*16 + h(0)

where h(n) is the nth hex digit (replace any letter digit with its decimal equivalent and start n at 0). For example, our number above in decimal is:

   dec = 9*164 + 8*163 + 10*162 + 3*16 + 15
   dec = 589824 + 32768  + 2560    + 48   + 15
   dec = 625215

As I said, this does not often need to be done, and if you need the decimal equivalent, feel free to find a calculator that does the conversion for you (or a web page; there are plenty of them on the internet).

Two hex digits represent one byte. Since much of memory is stored in one-byte increments, you will often see hex numbers written in groups of two to represent one byte per group. For our example (where we need to pad out with a zero the first group to make two digits):

   09 8A 3F

In C/C++ language, to indicate that a literal number is a hex number, use 0x or 0X in front of the number: 0x098A3F.

Here's an example of a small program that uses hex syntax for arithmetic:

  #include <stdio.h> // this is how we are able to call printf

  int main(){

    int hex1 = 0x1A; // 16 + 10 = 26
    int hex2 = 0xFF; // 15*16 + 15 = 255

    printf("hex1 in decimal is: %d\n", hex1); // the %d directive prints signed integers in decimal
    printf("hex2 in decimal is: %d\n", hex2);
    printf("The sum in hex is: 0x%x\n", hex1 + hex2); // the hex directive is %x (or %X if you want UPPERCASE LETTERS)

    return 0;
  }

Hex Color Codes

A lot of people's first introduction to writing numbers in hex isn't decompiling code or low-level system programming but dealing with color in art programs. For computers, color is generally represented as "24-bit color depth," which means that we're using 24 bits to represent the full RGB color. 

We do this by splitting the 24 bits into 8 bits for red, 8 bits for green, and 8 bits for blue.

Now, what numbers can you represent with 8 bits? 0 – 255!

And if you use two digits of hex, what numbers can you represent? Well, that's 0x00, which is 0 through 0xFF, which is

15*16 + 15

or 255.

So, in order to represent a 24-bit color, we can just write a hex number with six digits! Pure black is then 0x000000, pure white is 0xFFFFFF, and something like teal is 0x39a78e. If you've never played with hex representations of colors, check out Color Hex [link opens in new tab].
