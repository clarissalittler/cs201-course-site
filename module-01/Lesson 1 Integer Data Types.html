Integer Representation: Integer Data Types

As mentioned last module, there are two categories of integer types: signed and unsigned. The unsigned integer types can only represent positive and zero values. The signed integer types can represent positive, zero and negative values.

Signed values are said to have a sign (positive or negative) and a magnitude (ie. the size of the value). For example, -5 has a sign of - or negative and a magnitude of 5.

Below is a table of ranges for several unsigned and signed types. This is an abridged version from Figure 2.10 for 64-bit architecture. For unsigned, the minimum value is always 0.


Data Type Minimum Maximum
[signed] char -128 127
unsigned char 0 255
short -32,768 32,767
unsigned short 0 65,535
int -2,147,483,648 2,147,483,647
unsigned int 0 4,294,967,295

There are some facts to note. For signed, the max and min magnitudes are about 1/2 of what their unsigned counterparts are. Also note that for signed, the min is one bigger in magnitude than the max. We'll talk about why when we discuss signed encoding later.
