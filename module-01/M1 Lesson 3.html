Lesson 3: Numeric Representation

Information Storage

In this summary we will be covering the binary representation of data within a computer system. Data includes numbers, symbols (letters, digits, etc), graphics, audio, and other data. Throughout this course, we will concentrate on numeric data.

No matter what type of data, all data is represented by binary digits or bits. A bit can be one of two values: 0 or 1. Bits are stored in memory in a sequence of cells. Each 8 cells (1 byte) is addressable as a whole by the system. Therefore, the smallest unit of addressable memory is one byte (8 bits).

With the exception of one-byte data values, data is stored across multiple cells. We will talk about how this is done in the lecture on byte ordering.

The bit pattern is the series of bits across many cells that represent one data value. Since a user is not likely to know binary, this bit pattern will need to be translated into the actual value (numbers, letters, etc) that the user can understand. This is typically done in hardware.

Word and Data Sizes

The word size of a computer system is the size of a pointer. This is the number of bits needed to represent all addresses in memory. For a 64-bit architecture (system design), this means an address is represented by 64 bits. Because of this, the address size limits the maximum possible addressable memory locations. For a 32-bit architecture, this was about 4 billion, preventing 32-bit systems from having memory storage larger than 4 Gigabytes. Now, with the word size of most computer systems being 64-bit, the limitation on memory storage is 16 exabytes or about 1019 bytes. This is a huge amount of memory (it's about 4 billion times bigger than 4G memory) and will take quite a while before we will need to increase word size to increase memory.

Previously, the word size was also the given size of an int type on the system, but starting with 64-bit architecture, the int type is no longer the word size.

In C/C++ languages, all simple or primitive data types have a fixed size associated with them. The size depends on the word size of the architecture.

Datatypes like int can be different byte lengths on different architectures, and the sizeof() operator can be used to compensate for that:

int a[100];
int n_elements = sizeof(a) / sizeof(int);

Some things to note: char * is a pointer that holds an address, so it needs to be the word size, which, for a 64-bit architecture, is 8 bytes (64/8 = 8).

There are signed and unsigned versions of integer numbers. The signed versions can represent both positive, zero, and negative. The unsigned versions can only represent positive and zero. They are typically the same size but have different ranges of values. We'll talk at length about signed and unsigned integer types next module. Floating point number types (float and double) are always signed types. We'll talk about floating point representation in Module 3.
