Module 4 Overview

In Module 4: x86-64 Assembly — Control, Functions, and Data, you will continue writing assembly programs by hand while learning how the processor handles control flow, function calls, and structured data. This module builds directly on the fundamentals from Module 3.

We begin with condition codes, comparisons, and jump instructions, then use them to implement if statements, loops, and switch statements. Next, we write functions in assembly: learning the calling convention, the stack frame layout, and recursive function calls. We also look at how arrays and structs are laid out in memory, examine buffer overflow vulnerabilities and their defenses, and conclude with floating point operations. Each topic follows the "write it by hand first, then see how gcc does it" approach.

Learning objectives

At the end of this learning module, you will be able to:

  1. Explain how condition codes are set by cmp and test, and use them with set and jump instructions
  2. Write assembly programs that implement if statements, loops, and switch statements
  3. Write assembly functions using the System V AMD64 calling convention (call, ret, argument registers)
  4. Identify the components of a stack frame and explain the function prologue/epilogue
  5. Explain callee-saved vs. caller-saved registers and use them correctly in hand-written functions
  6. Write recursive assembly functions
  7. Describe how arrays and structures are laid out in memory, including alignment and padding
  8. Explain buffer overflow attacks and the defenses against them (ASLR, stack canaries, NX bit)
  9. Describe how floating point values are handled using XMM registers and SSE2 instructions

Learning activities

You are expected to do the following tasks in order to achieve the above objectives:

  1. Complete the reading listed below
  2. Read Lessons 1 to 9, work through the hand-written assembly examples, and complete the embedded practice problems
