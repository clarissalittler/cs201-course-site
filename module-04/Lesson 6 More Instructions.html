Lesson 6: Move Instructions

Move Instructions

We are now ready to talk about one of the most common instructions in assembly code: the series of mov instructions. These instructions move data from one memory/register location to another memory/register location. Unlike high-level code, the CPU explicitly needs to be told where to move data. Data can be moved from register to register, from memory location to register, and from register to memory location, but it cannot move from one memory location to another memory location. If you need to move from one memory location to another, you need to do this in two steps using a register: memory 1 -> register; then register -> memory 2.

Each mov instruction ends with one of the size indicators mentioned above. For example, if you need to move an int size amount of data, you would use the movl command. The first operand is the source or what is to be moved. The second operand is the destination or where the operand is to be moved to. Note that the first or source operand can also be an immediate. Examples of using this command are below:

Example 1:

Move a 32 bit hex value of 0x56 to the register ebx.

movl $0x56,%ebx

Example 2:

Move a 64 bit value from the register rax to the register rcx. Note that we need to use the appropriate names for the given size.

movq %rax,%rcx

Example 3:

Move a 16 bit value from the memory address stored in rcx to the register bx. Note that the first operand is an address, not a value, so the register name needs to be 64 bit. But the register that the value is moved to is 16 bit, so we use the 16 bit name.

movw (%rcx),%bx

There are other examples in the book that you can look at. Also, look at some disassembled code of your own written C code and see if you can guess what each instruction is doing.

Also available are instructions that move data while changing the size of the value being moved. I won't be requiring you to know these instructions. You can read about these in the textbook under 3.4.2 Data Movement Instructions.
