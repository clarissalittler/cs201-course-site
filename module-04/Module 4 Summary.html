Module 4 Summary

Wrap up

In this module, you wrote assembly programs with control flow, functions, and structured data — completing the full arc from a 3-line exit program in Module 3 to recursive functions and complex data structures. You also examined buffer overflow vulnerabilities and floating point operations.

Key Takeaways

  • Condition codes (CF, ZF, SF, OF) are set by arithmetic and comparison instructions. The cmp and test instructions set flags without storing a result. Jump instructions and conditional moves read these flags to implement branching.
  • All control flow — if statements, loops, and switch statements — is built from compare-and-jump patterns. Switch statements with many cases use jump tables for O(1) dispatch.
  • Functions in assembly are defined by convention, not hardware. The System V AMD64 calling convention specifies argument registers (%rdi, %rsi, ...), the return register (%rax), and which registers are callee-saved vs. caller-saved.
  • A function's stack frame holds the return address, saved registers, and local variables. The prologue (push %rbp / mov %rsp, %rbp / sub) and epilogue (leave / ret) manage this frame.
  • Arrays use scaled-index addressing (base + index × size). Structs use fixed offsets with alignment padding. Reordering struct fields from largest to smallest minimizes wasted space.
  • Buffer overflows exploit C's lack of bounds checking. Defenses include ASLR (randomizing addresses), stack canaries (detecting corruption), and the NX bit (preventing code execution from the stack).
  • Floating point uses a separate set of 16 XMM registers and SSE2 instructions (movsd, addsd, mulsd, etc.). FP arguments pass in %xmm0–%xmm7, separately from integer arguments.
