Lesson 7: Push and Pop

Push and Pop Instructions

Another type of movement instruction is push and pop. The main memory location where local variables are stored is called a program stack. A stack is a data structure that acts like a stack of plates. You push onto the stack by putting a plate on top of the stack and pop off the stack when you take a plate from the top of the stack. Only the top of the stack is accessed at any time.

Basically, you can look at these instructions as one of several ways the compiler can fetch and retrieve data from memory. gcc uses it mainly to keep values in registers from being overwritten between function calls.

A push instruction takes one operand, which is either a memory location, a register, or an immediate value. This is then placed on the 'top' of the current program stack. The instruction usually takes a size indicator. A pop command will take the value at the top of the stack and put it into the operand's location. Again, this is either a register or memory location.

We'll talk more about the 'program stack' and using push and pop commands in week 6.
