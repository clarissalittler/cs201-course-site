Pointwise Bit Operators

Students are familiar with boolean operators && for AND and || for OR operations. These work at the numeric level in that they interpret a conditional or number to be 1 (TRUE) or 0 (FALSE).

There are boolean operators that work on the bit pattern itself. These are called bit-wise boolean operators because they perform boolean operations to each bit one at a time as opposed to the entire number. The following is a table of these bit-wise boolean operators:

Operator Name Example
& bit-wise AND 1010   &   1001   => 1000
| bit-wise OR 1010   |   1001   => 1011
~ complement or negation ~ 1011   => 0100
^ eXclusive OR 1010   ^   1001   => 0011

Note that each binary operator compares each bit with the corresponding bit in the same position. For bit-wise AND, if both bits are 1 then the result is 1 otherwise it is 0. For bit-wise OR, if both bits are 0 then the result is 0 otherwise it is 1. For exclusive OR, if both bits are the same the result is 0, otherwise the result is 1. The complement or negation operator flips the bit's value. If the bit is 1 then the result is 0; if the bit is 0 the result is 1.

Note that the bit-wise & and | use only one symbol instead of two. This is a common mistake, so be aware of it.

Shift Operators

Another set of bit-wise operators are bit shift operators. These will move the bit pattern over by some given amount of positions for a given number.

The left shift operator ( num << i ) moves each bit in num to the left by i amount. For example, if num=10010011 and i=2, the resulting value will be:

 10010011 => 01001100

Note that all left bits that don't fit into the number are just ignored, and the new right bits are filled with 0's.

The right shift operator ( num >> i) will shift the bit pattern of num by i amount to the right. For example, if num is an unsigned char equal to 10010011 and i=2:

 10010011 => 00100100

For all integers, the rightmost bits are ignored. For unsigned integers, the bits on the left are filled in with 0's. This is called a logical shift. For signed integers, the behavior is different in that the left side is filled with whatever the original leftmost bit is (1 or 0). This is called an arithmetic shift. So, for our example, if num above is a signed char:

 10010011 => 11100100

Another example:

 01100010 => 00011000

Note that the bit that is filled in is whatever the leftmost bit happens to be. We'll discuss next module why this is so when we talk about how signed integers are represented in memory.
