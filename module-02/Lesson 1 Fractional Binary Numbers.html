Introduction

In this summary, we will look at floating point numbers. We will start by looking at how floating point is represented as binary numbers. Then, we will discuss how they are represented in computer memory. Other topics covered are rounding floating point numbers and problems encountered in doing floating point operations.

In C/C++, there are only two commonly used floating point types: float, which is 32 bit, and double, which is 64 bit. The float type is considered to be single-precision, and the double type is considered to be double-precision. Precision is how many digits there are in a floating point number; the more digits, the higher the precision.

Section 2.4.1: Fractional Binary Numbers

Fractional binary numbers have similar notation to decimal numbers. Basically, the fractional part is separated by a period:

  1011.101, 0.1101, 0.0001

the digits after the decimal point are calculated by using 1/2n in place of 2n starting at 1/2 for the first digit after the decimal. So, for the number 1011.101:

   1*23 + 0*22 + 1*21 + 1*20 + 1*1/21 + 0*1/22 + 1*1/23

   8 + 0 + 2 + 1 = 11
   1/2 + 0 + 1/8 = 5/8 = .625

   so 1011.101 is 11.625 in decimal

Fractional binary numbers can be represented in scientific notation the same as decimal:

   1011.101  =>  1.011101 x 23

Note that technically only the mantissa part (the digits to the left of the 23 part) is binary; the base and exponent are typically written in decimal to prevent confusion.

From Fractional Binary to Fully Decimal Number

The above isn't quite in decimal format. To go from fractional binary to full decimal, we need to convert the mantissa to decimal and then multiply by the 2 base exponent part. For example:

    1.0011 x 213
    1.0011 => 1*20 + 1*2-3 + 1*2-4 = 1.1875
    1.1875 x 213 = 9728
    9.728 x 103
