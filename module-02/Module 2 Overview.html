Module 2 Overview

In Module 2: Floating Point, we will explore how computers represent numbers that have fractional parts. Unlike integers, which can be represented exactly in binary, floating point numbers require an approximation scheme that trades off range, precision, and efficiency. We will study the dominant standard (IEEE 754) and also examine an alternative representation called posits.

This module contains 6 lessons:

  1. Lesson 1: Fractional Binary Numbers — How to represent fractions in binary using a radix point, and converting between binary and decimal fractions.
  2. Lesson 2: IEEE 754 Floating Point Encoding Scheme — The structure of IEEE 754: sign bit, exponent field, and fraction (mantissa) field for single and double precision.
  3. Lesson 3: Floating Point Examples — Worked examples encoding and decoding IEEE 754 floating point values, including normalized, denormalized, and special values.
  4. Lesson 4: Floating Point Rounding — The four IEEE rounding modes (round-to-even, round toward zero, round down, round up) and why round-to-even is the default.
  5. Lesson 5: Floating Point Operations — How addition, multiplication, and other operations work on floating point numbers, and the precision and overflow problems that can arise.
  6. Lesson 6: Posit Number System — An alternative to IEEE 754 developed by John Gustafson, featuring variable-length regime bits and tapered precision.

Learning Objectives

At the end of this learning module, you will be able to:

  1. Represent fractional numbers in binary using a radix point
  2. Encode and decode IEEE 754 floating point values in both single precision (32-bit) and double precision (64-bit) formats
  3. Identify and explain the three categories of IEEE 754 values: normalized, denormalized, and special values (infinity, NaN)
  4. Apply the four IEEE rounding modes and explain why round-to-even is preferred
  5. Recognize precision loss, overflow, and other problems that arise in floating point arithmetic
  6. Describe the posit number system and compare it with IEEE 754 representation
