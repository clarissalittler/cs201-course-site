Module 9 Overview

In Module 9: Memory Systems, you will look at the memory hierarchy. The memory hierarchy consists of many parts spread out throughout the computer's hardware. A list of parts includes:

  • Internal main memory
  • CPU chip memory (registers and memory caches)
  • Disk storage

This module will focus on those aspects of hardware that might be of interest to programmers. There are a lot of detailed descriptions of how the hardware works.  We will selectively look at some sections in this Chapter.

Learning objectives

At the end of this learning module, you will be able to:

  1. Describe basic storage technologies, SRAM memory, DRAM memory, ROM memory, and rotating and solid state disks
  2. Describe how the storage technologies are organized into hierarchies.
  3. Interpret how the cache memories act as staging areas between the CPU and main memory.
  4. Analyze your C programs for locality and learn about techniques for improving the locality in your programs. 
  5. Characterize the performance of the memory hierarchy on a particular machine as a “memory mountain” that shows read access times as a function of locality.

Learning activities

You are expected to do the following tasks in order to achieve the above objectives:

    1. Read Chapter 6, sections 6.1 to 6.4 in textbook
    2. Read Lessons 1 to 6 and complete the embedded practice problems
