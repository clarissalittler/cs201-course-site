Lesson 3: Accessing Main Memory

Accessing Main Memory

Buses carry data (and addresses) back and forth from the CPU to main memory. A bus is a collection of parallel wires inside the computer bundled in ribbon-like connectors. There are many of them but we will discuss just those that connect the CPU to main memory.

There are two buses that connect the CPU to main memory: One (system bus) is from the CPU to a controller device called an I/O Bridge. The other (memory bus) goes from the I/O Bridge to the main memory. An I/O Bridge is just a device that routes requests from the CPU to the correct location: main memory, disk drive and so on. There are other buses called I/O buses that connect peripherals with the CPU: graphics cards, monitors, mice, keyboards and external drives.

Whenever the CPU and main memory need to send data to each other, they set up a bus transaction. This is a series of steps that transfer data in one direction. A read transaction transfers data from main memory to the CPU; a write transaction transfers data from the CPU to main memory.
