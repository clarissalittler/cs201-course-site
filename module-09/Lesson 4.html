Lesson 4: Disk Storage

Disk Storage

Disks are the name given to non-volatile storage devices that are read or written to and hold a large amount of information. Nowadays, a disk is not likely to be seen in most of these devices as many disks use flash memory to store data (SSD's).

What to Read in this Section

Albeit actual disk drives with disks (hard disk drives or HDD) are still around, they are increasingly becoming replaced by solid state drives or SSD's. So I'm putting less emphasis on any hard disk information in this section. In fact, I won't ask questions on disk geometry, disk capacity or disk operations. So you can skip these parts unless your are interested in learn more about hard disks.

The following list below lists basically what I want you to know about this section (6.1.2).

  • The advantages of using SSD's over HDD's:
    1. Faster access times
    2. Not prone to 'head crashes' so more robust
    3. Uses less power.
  • The disadvantages of using SSD's over HDD's
    1. more expensive per byte. This makes capacity less for the same price.
    2. May wear out faster.

In general, when it comes to hardware technologies there's always a tradoff between quality/performance and cost.

From figure 6.16, you can see trends in access time for various technologies from 1980's to 2015.

  • Hard Disk has always had dramatically worse performance than memory. It is slower than memory by a factor 1,000,000 and its not getting any better. Hard disks are mechanical in nature and have physical limitations in speed that is not feasible to overcome.
  • Main memory started out at the same level as CPU but has not kept up with CPU performance. This is called the memory gap and one of the reasons why accessing main memory is now done less often in programs. gcc tends to favor using registers whenever possible for this reason.
  • There's an ever widening gap between CPU, main memory and disk. This might be mitigated when systems start adopting SSD's or we come up with better main memory technology (maybe better memory cache management).

The main take away is that main memory is slower to read/write to than registers on the CPU but getting anything off a disk is even slower (from one hundred thousand up to a million times slower than memory).
