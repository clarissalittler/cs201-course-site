Module 5 Overview

In Module 5: Processes, Signals, and Forking, we move from the instruction level up to the operating system level. We will study how the operating system manages running programs through processes and how it handles unexpected events through exceptional control flow.

We start by discussing control flow and exceptional control flow. The sole job of the CPU is to process one instruction after another, but sometimes events -- both hardware and software -- cause an abrupt change in that flow. We will talk about exceptions, what causes them, and how they are handled. From there, we learn about processes: what they are, how the operating system creates and manages them, and how to use system calls such as fork, exec, and wait to create and coordinate child processes. The module concludes with signal handling: how signals work as software-level exceptions, how to write signal handlers, and how to block signals safely from within your C code.

This module consolidates what were previously two separate modules (Processes and Signal Handling) into one comprehensive unit.

Learning objectives

At the end of this learning module, you will be able to:

  1. Explain control flow and exceptional control flow
  2. Define the four classes of exceptions (interrupts, traps, faults, and aborts)
  3. Describe what a process is and how processes are managed by the operating system
  4. Use fork to create child processes and predict the behavior of programs that use it
  5. Use exec to replace a process image and wait/waitpid to synchronize parent and child processes
  6. Explain how signals and signal handlers work
  7. Write signal handlers and install them using signal or sigaction
  8. Block and unblock signals from your C code using the appropriate library functions
