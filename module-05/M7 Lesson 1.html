Flow Control

The sole job of the CPU is to process one instruction after another. Going from one instruction to another is called control transfer. A set of control transfers is called a flow of control or control flow.

Normal control flow is typically moving from one instruction to another along with any jumps. This is usually based on the 'state' of the internal variables. Sometimes this flow control needs to be interrupted by outside events.

For example, a hardware timer goes off at regular intervals and must be dealt with. A program had requested data from a disk and needs to be notified that the data is ready. Parent processes that create child processes must be notified when their children terminate.

In these cases the operating system makes some abrupt changes to the normal flow control of the program being run. This is called exceptional flow control (EFC). It is exceptional in that it would not be the normal flow control for the program if some event hadn't happened to interrupt this normal flow.
