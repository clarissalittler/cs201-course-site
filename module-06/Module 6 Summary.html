Module 6 Summary

Wrap up

In this module, we covered more object code related to Procedure, Data Structures, Security, and Floating Point numbers.

The topics included:

  • Procedures (another name for functions): how they are done in assembly code.
  • The program stack in detail: how it is managed and what is stored there.
  • Structures and how byte alignment can make them seem bigger than they are.
  • Security issues that arise when buffer overflows happen.
  • Floating point numbers, their registers, and circuitry used for arithmetic, logic, and other tasks.

Summary

  • Procedures are what C language calls functions. These handy programming structures need to be fully implemented in assembly language as there are few built-in mechanisms for procedures in assembly.
  • We saw how complex data structures, built from many values, are maintained in assembly code.
  • Over the years, buffer overflow attacks have become very popular ways of breaking into systems. So recently, compilers and hardware designers have become more proactive in stopping buffer overflows.
  • Floating point numbers are stored in a separate set of registers from the integer registers we have been working with. There are 16 registers used for both single (float) and double precision.
