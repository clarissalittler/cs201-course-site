Module 6 Overview

In this Module 6: Threads and Synchronization, you will learn about:

  • Threads: what they are, how they differ from processes, and why they are a fundamental building block of concurrent programs.
  • The POSIX threads (pthreads) library: how to create, manage, and join threads in C programs.
  • Race conditions: what happens when multiple threads access shared data without coordination, and why the results are unpredictable.
  • Mutexes: the primary synchronization primitive for protecting critical sections of code.
  • Condition variables and semaphores: more advanced synchronization tools for coordinating thread activity.
  • Classic concurrency problems: the Dining Philosophers problem and the Reader-Writer problem, which illustrate the challenges of concurrent programming.

Background

In earlier modules, you learned how to create new processes using fork() and how processes communicate using signals. Processes are heavyweight: each fork creates a complete copy of the address space. Threads offer a lighter-weight alternative. Multiple threads within the same process share the same address space (code, heap, global variables) but each has its own stack and set of registers. This makes threads cheaper to create and allows them to communicate through shared memory. However, shared memory introduces new problems: when multiple threads read and write the same data simultaneously, the results can be unpredictable and incorrect. This module teaches you how to write correct concurrent programs by using synchronization primitives.

Learning Objectives

At the end of this learning module, you will be able to:

  1. Explain the difference between threads and processes, including what is shared and what is private to each thread.
  2. Create and join POSIX threads using pthread_create() and pthread_join(), passing arguments and receiving return values.
  3. Identify race conditions in concurrent code and explain why they produce incorrect results.
  4. Use mutexes (pthread_mutex_t) to protect critical sections and prevent race conditions.
  5. Use condition variables (pthread_cond_t) and semaphores (sem_t) to coordinate threads that must wait for specific conditions.
  6. Describe classic concurrency problems (Dining Philosophers, Reader-Writer) and implement solutions that avoid deadlock and starvation.
  7. Identify and avoid common pitfalls in concurrent programming, including deadlock, starvation, and data races.

Learning Activities

You are expected to do the following tasks in order to achieve the above objectives:

  1. Read Lessons 1 through 6 and study the code examples carefully. Type them in and run them yourself.
  2. Compile and experiment with the provided C programs. Observe how race conditions produce different results on each run.
