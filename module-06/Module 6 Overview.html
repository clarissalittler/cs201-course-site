Module 6 Overview

In this Module 6: Object Code - Procedures and Floating Point, you will learn about:

  • Procedures (another name for functions): how are they done in assembly code.
  • The program stack in detail: how it is managed and what is stored there.
  • Structures and how byte alignment can make them seem bigger than they are.
  • Security issues that arise when buffer overflows happen.
  • Floating point numbers, their registers and circuitry used for arithmetic, logic and other tasks.

Learning objectives

At the end of this learning module, you will be able to:

  1. Explain basic functionality of the pairs: push / pop and call / ret
  2. Identify the different components of a stack (return address, arguments, saved registers, local variables)
  3. Explain the difference between callee and caller save registers
  4. Explain how a stack permits functions to be called recursively / re-entrant 
  5. Explain how floating point numbers are handled in assembly code

Learning activities

You are expected to do the following tasks in order to achieve the above objectives:

    1. Read Chapter 3, sections 3.7, 3.9, and 3.10 in textbook
    2. Read Lessons 1 to 4 and complete the embedded practice problems
    3. Complete Discussion 6 
    4. Take Quiz 6
    5. Review additional resources in the module summary
