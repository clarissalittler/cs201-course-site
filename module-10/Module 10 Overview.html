Module 10: Kernel Organization

In Module 10: Kernel Organization, you will explore the heart of every operating system: the kernel. Throughout this course, you have been using kernel services every time you called fork(), sent a signal, created a thread, opened a socket, or allocated memory. Now we step back and examine how the kernel itself is designed and organized.

This module covers the two major approaches to kernel architecture — monolithic kernels and microkernels — along with their tradeoffs and modern variations. You will learn:

  • What a kernel does and how it mediates between user programs and hardware
  • How monolithic kernels like Linux organize all OS services into a single privileged program
  • How microkernels take the opposite approach, running most services in user space
  • How modern systems blend these approaches and what tradeoffs drive those decisions

Learning objectives

At the end of this learning module, you will be able to:

  1. Explain the role of a kernel, including how it manages processes, memory, devices, and file systems on behalf of user programs
  2. Compare and contrast monolithic and microkernel designs, identifying the key architectural differences and their consequences
  3. Describe system call dispatch mechanisms, including how the CPU transitions between user mode and kernel mode
  4. Evaluate tradeoffs between kernel architectures in terms of performance, reliability, security, and verifiability

Learning activities

You are expected to do the following tasks in order to achieve the above objectives:

  1. Read Lessons 1 through 4 and complete the embedded practice problems
  2. Complete Discussion 10

Prerequisites

This module builds on concepts from throughout the course. You should be comfortable with:

  • C programming and system calls (fork(), exec(), wait(), open(), read(), write())
  • Processes, signals, and threads (Modules 5-7)
  • Sockets and inter-process communication (Module 8)
  • Memory hierarchy and virtual memory (Module 9)
